Always check shadcn/ui documentation BEFORE creating any UI component. Never create custom components when shadcn has an official equivalent.

**Why:** The user has emphasized this multiple times. The project uses shadcn/ui as the component library and custom components create inconsistency.

**How to apply:** Before writing any new UI component or modifying existing ones, use the `shadcn` skill to search if a matching component exists. This applies to every UI-related task, not just when it's obvious.
